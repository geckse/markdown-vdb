Session 3 (Subtask 3-1 — Cluster Enclosure Spheres):
- Added cluster enclosure spheres to GraphView.svelte
  - Imported `* as THREE from 'three'` for direct Three.js scene manipulation
  - New state variables:
    - clusterMeshGroup (THREE.Group) — container for all sphere meshes
    - engineTickCount — throttle counter for tick-based updates
    - clusterLabels — screen-projected label positions for HTML overlay
  - clearClusterMeshes(): disposes all geometries/materials, removes group from scene
  - updateClusterSpheres(): computes enclosing sphere per cluster_id
    - Centroid = average of node positions in cluster
    - Radius = max distance from centroid to any node + 20 padding (min 10)
    - Fill mesh: SphereGeometry + MeshLambertMaterial (opacity 0.06, BackSide, depthWrite false)
    - Wire mesh: SphereGeometry + MeshBasicMaterial (wireframe, opacity 0.12, depthWrite false)
    - Color matches cluster palette (CLUSTER_COLORS[clusterId % 12])
  - projectToScreen(): 3D world → 2D screen coordinate projection via camera
  - updateClusterLabelPositions(): recompute label screen positions during camera movement
  - recomputeLiveCentroids(): updates clusterCentroids map for attraction force targets
  - handleEngineTick(): throttled (every ~30 ticks) sphere + centroid updates during simulation
  - handleEngineStop(): final sphere update when simulation completes
  - updateClusterSphereMeshPositions(): repositions meshes during simulation (delegates to full rebuild)
  - onEngineTick/onEngineStop callbacks registered in onMount
  - Coloring mode subscription updated: cluster mode → updateClusterSpheres(), other modes → clearClusterMeshes()
  - Proper cleanup in onDestroy: clearClusterMeshes() before graph._destructor()
- Template additions:
  - Cluster label HTML overlay: positioned absolutely, pointer-events: none, text-shadow for readability
  - Labels only visible in cluster coloring mode and when label.visible (in front of camera)
- CSS additions:
  - .cluster-label: absolute positioning, translate(-50%, -50%) centering, text-shadow,
    opacity 0.7, z-index 5, font-weight 500

Session 4 (Subtask 3-3 — Selection Dimming & Hub Node Glow):
- Added selection dimming to GraphView.svelte
  - computeNeighborSet(): computes set of neighbor node IDs from graph edges
    - Includes the selected node itself in the set
    - Traverses all links, checks source/target against selected node ID
  - applySelectionDimming(): two-phase approach since nodeOpacity/linkOpacity are static
    - Selection active:
      - graph.nodeOpacity(0.15) — dims all default-rendered nodes globally
      - graph.linkOpacity(0.05) — dims all links globally
      - graph.nodeThreeObject() override: creates custom THREE.Group objects for
        selected+neighbor nodes at 0.9 opacity (overriding the global dim)
      - graph.linkMaterial() override: creates LineBasicMaterial at 0.6 opacity
        for neighbor links (overriding the global dim)
    - No selection:
      - Restores nodeOpacity(0.9), linkOpacity(0.4)
      - Restores nodeThreeObject to hub-only mode
      - Clears linkMaterial override (null)
  - Updated graphSelectedNode subscription to call computeNeighborSet() + applySelectionDimming()
- Added hub node glow
  - createHubNodeObject(): creates THREE.Group with:
    - SphereGeometry(radius, 12, 8) + MeshLambertMaterial (node color, opacity 0.9)
    - PointLight (node color, intensity 0.5 or 0.1 if dimmed, distance = radius * 5)
    - Radius computed as Math.cbrt(node.val) * 4 (matches 3d-force-graph default)
  - nodeThreeObject accessor: returns custom Group for degree >= 5, false for < 5
  - Hub objects get dimmed opacity during selection dimming (non-neighbor hubs at 0.15)
- Added node labels on hover
  - nodeLabel accessor: calls nodeTooltipHtml() from graph-3d-bridge
  - Shows path, cluster label (when present), heading (chunk mode)
- Added edge labels on hover
  - linkLabel accessor: calls edgeTooltipHtml() from graph-3d-bridge
  - Shows relationship type, strength percentage, context excerpt (truncated 120 chars)
- New imports: nodeTooltipHtml, edgeTooltipHtml from graph-3d-bridge
- New state: neighborSet (Set<string>)
- TypeScript note: nodeThreeObject returning false requires `as any` cast since
  the type expects Object3D, but runtime handles falsy values as "use default sphere"
